[原]Java之泛型剖析

Posted on 2017-06-17 
 泛型程序设计意味着编写的代码可以被很多不同类型的对象所重用。泛型提供了一个类型参数用来指示元素的类型。比如AarryList:
 ArrayList&lt;Sting&gt; strs = new ArrayList&lt;String&gt;();(根据多态性，前面的ArrayList&lt;String&gt;可以换成List&lt;String&gt;)。这样使代码有更好的可读性。
`</pre>

一个泛型类就是具有一个或多个类型变量的类。下面是一个简单的泛型类：

<pre class="prettyprint">` <span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> Result&lt;T&gt; {
    <span class="hljs-keyword">private</span> T first;
    <span class="hljs-keyword">private</span> T second;

    <span class="hljs-keyword">public</span> T <span class="hljs-title">getFirst</span>() {
        <span class="hljs-keyword">return</span> first;
    }

    <span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">setFirst</span>(T first) {
        <span class="hljs-keyword">this</span>.first = first;
    }

    <span class="hljs-keyword">public</span> T <span class="hljs-title">getSecond</span>() {
        <span class="hljs-keyword">return</span> second;
    }

    <span class="hljs-keyword">public</span> <span class="hljs-keyword">void</span> <span class="hljs-title">setSecond</span>(T second) {
        <span class="hljs-keyword">this</span>.second = second;
    }

    <span class="hljs-keyword">public</span> <span class="hljs-keyword">static</span> &lt;T&gt; T <span class="hljs-title">getObj</span>(Class&lt;T&gt; cls) throws IllegalAccessException, InstantiationException {
        <span class="hljs-keyword">return</span> cls.newInstance();
    }
}`</pre>

<pre>`Result类中引入类型变量T，用尖括号(&lt;&gt;)括起来，并放在类后面。泛型类可以有多个类型变量，其中第一个域和第二个域可以有不同的类型：public class Result&lt;T, U&gt;{....}。类定义中的类型变量指定方法的返回类型以及域和局部变量的名字。
上面的例子中还定义了一个泛型方法getObj()。当然泛型方法也可以在普通类中定义。
`</pre>

## 类型变量的限定

<pre>`有时，类或方法需要对类型加以约束
`</pre>

<pre class="prettyprint">`<span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> ArrayTest {
    <span class="hljs-keyword">public</span> <span class="hljs-keyword">static</span> &lt;T extends Comparable&gt; T <span class="hljs-title">min</span>(T[] a) {
        <span class="hljs-keyword">if</span> (a == <span class="hljs-keyword">null</span> || a.length == <span class="hljs-number">0</span>) {
            <span class="hljs-keyword">return</span> <span class="hljs-keyword">null</span>;
        }
        T temp = a[<span class="hljs-number">0</span>];
        <span class="hljs-keyword">for</span> (<span class="hljs-keyword">int</span> i = <span class="hljs-number">1</span>; i &lt; a.length; ++i) {
            <span class="hljs-keyword">if</span> (temp.compareTo(a[i]) &gt; <span class="hljs-number">0</span>) {
                temp = a[i];
            }
        }
        <span class="hljs-keyword">return</span> temp;
    }
}`</pre>

<pre>`这里对T的类型限制为实现了Comparabe接口的类。之所以这样做是因为，只有实现了Comparabe接口的类才会实现compareTo()方法。程序才能正确编译执行。
（这里需要注意的是Comparabe接口本身就是一个泛型类型）。
&lt;T extends BoundingType&gt;表示T应该是绑定类型（BoundingType）的子类型。T和绑定的类型可以是类也可以是接口。（均使用extends关键字）。
一个类型变量或通配符可以有多个限定，例如：
T extends Comparable &amp; Serializable
限定类型用“&amp;”分隔，而逗号用来分隔类型变量。注意点：Java的继承中，可以根据需要拥有多个接口超类型，但限定至多有一个类。如果一个类作为限定，它必须是限定列表中的第一个。
`</pre>

## 泛型的擦除

<pre>`虚拟机没有泛型类型对象，所有类型都属于普通类。当定义一个泛型类型时，都自动提供了一个相应的原始类型。原始类型就是删去类型参数后的泛型名。擦除类型变量，并替换为限定类型。
原始类型用第一个限定的类型变量来替换，如果没有给定限定就用Object替换。比如上面的例子，因为T无类型限制，所以会被用Object代替。例子如下：
`</pre>

<pre class="prettyprint">`<span class="hljs-keyword">public</span> <span class="hljs-class"><span class="hljs-keyword">class</span> <span class="hljs-title">ResultSecond</span>&lt;<span class="hljs-title">T</span> <span class="hljs-inheritance"><span class="hljs-keyword">extends</span></span> <span class="hljs-title">Comparable</span> &amp; <span class="hljs-title">Serializable</span>&gt; <span class="hljs-inheritance"><span class="hljs-keyword">implements</span></span> <span class="hljs-title">Serializable</span> {</span>
    <span class="hljs-keyword">private</span> T first;
    <span class="hljs-keyword">private</span> T second;

    <span class="hljs-keyword">public</span> ResultSecond(T first, T second) {
        <span class="hljs-keyword">if</span> (first.compareTo(second) &gt; <span class="hljs-number">0</span>) {
            <span class="hljs-keyword">this</span>.first = first;
            <span class="hljs-keyword">this</span>.second = second;
        } <span class="hljs-keyword">else</span> {
            <span class="hljs-keyword">this</span>.first = second;
            <span class="hljs-keyword">this</span>.second = first;
        }
    }
}`</pre>

原始类型如下:

<pre class="prettyprint">`<span class="hljs-keyword">public</span> <span class="hljs-keyword">class</span> ResultSecond {
    <span class="hljs-keyword">private</span> Comparable first;
    <span class="hljs-keyword">private</span> Comparable second;

    <span class="hljs-keyword">public</span> <span class="hljs-title">ResultSecond</span>(Comparable first, Comparable second) {
        <span class="hljs-keyword">if</span> (first.compareTo(second) &gt; <span class="hljs-number">0</span>) {
            <span class="hljs-keyword">this</span>.first = first;
            <span class="hljs-keyword">this</span>.second = second;
        } <span class="hljs-keyword">else</span> {
            <span class="hljs-keyword">this</span>.first = second;
            <span class="hljs-keyword">this</span>.second = first;
        }
    }
}`</pre>

## 几个注意点

<pre>`1、不能用基本类型实例化类型参数。
因此没有Result&lt;int&gt;,只有Result&lt;Integer&gt;。原因是类型擦除。擦除后，Result类含有Object类型的域，而Object不能存储int值。
2、运行时类型查询只适用于原始类型。
虚拟机中的对象总有一个特定的非泛型类型。因此所有的类型检查只产生原始类型。例如：if (a instanceof Result&lt;T&gt;)，仅仅只是测试a是否是任意类型的一个Result。
3、不能创建参数化类型的数组。
不能实例化参数化类型的数组。
例如：Result&lt;String&gt;[] result = new Result&lt;String&gt;[5];这是不被允许的，因为擦除之后，result类型就是Result[]类型。
这里只是不允许创建数组，但是声明为Result&lt;String&gt;[]的变量仍是合法的。不过不能用new Result&lt;String&gt;[10]初始化这个变量。
4、不能实例化类型变量。不能使用new T(...)，new T[...]。
5、泛型类的静态上下文类型变量无效。即不能在静态域或方法中引用类型变量。
6、不能抛出或捕获泛型类的实例
7、注意擦除后的冲突。
